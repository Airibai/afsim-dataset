# 平台移动快捷键

AFSIM 还提供了快捷键，用于向一组平台发出方向命令，提高同时控制多个平台的效率：

转向航向（默认键 $= \mathsf { H }$ ）：平台将尝试转向指定的航向，使用其移动器的默认加速度。  
到达位置（默认键 $= { \sf G }$ ）：平台将尝试移动到指定位置，保持当前高度，并使用其移动器的默认加速度。

# 重要提示

移动器要求：平台必须有一个移动器才能控制其移动。移动器决定平台如何响应每个命令，并且某些移动器可能不支持所有命令。  
命令支持：平台执行这些命令的能力取决于其移动器的能力，不同类型的平台之间可能会有所不同。

# 操纵杆的基本概念

操纵杆（Joystick）是一种输入设备，通常用于控制飞机的主要控制装置。它通过在基座上的旋转来报告其角度或方向，从而控制设备。这种设备在许多民用和军用飞机的驾驶舱中是主要的控制装置。

# 控制器映射概述

AFSIM 的 Joystick 插件支持多种控制器，这些映射定义在存储于 resources/data 目录下的XML 文件中。以下是 Warlock 附带的控制器映射：

# XML 文件结构

XML 文件包含一个 devicelist，其中可以包含一个或多个设备。每个设备定义了插件的控制方案。建议每个文件只包含一个设备。

```txt
<device1> <device name="My new controller"> ... </device> <device name="My other controller"> ... </device> 
```

```twig
</devicelist> 
```

设备定义

id_string(s): 定义控制器在操作系统中的识别方式。可以使用通配符。一个控制方案可以使用多个控制器（例如，操纵杆和油门），在这种情况下，应在 id_string 上定义子设备。

单个子设备示例：

```txt
<device name="My single controller"> <id_string name="Controller*"/> ... </device> 
```

多个子设备示例：

```xml
<device name="My Stick and Throttle">
    <id_string name="Stick*" subdevice="0"/>
    <id_string name="Throttle*" subdevice="1"/>
    ...
</device> 
```

values: 定义应用程序与控制器之间的绑定。值的名称来自插件。值可以包含各种控制机制：  
axis: 返回值范围为[-1,1]的控制。  
button: 返回二进制值的控制，默认返回 0，按下时返回 1。  
toggle_button: 循环通过状态，默认在 0 和 1 之间循环。  
hat: 两轴控制，每个轴的默认范围为[-1,1]。  
state_key: 将键盘输入视为按钮。  
toggle_key: 将键盘输入视为 toggle_button。

示例：

```xml
<value name="buttonActivate_on_press">
    <button name="0"/>
</value> 
```

使用两个（肩部）按钮模拟轴或帽轴：

```xml
<value name="rudder">
    <button name="0" value="0,1"/>
    <button name="1" value="0,-1"/>
</value> 
```

# 5.2.2.5.4. 六自由度数据 P6Dof Data - Warlock

# P6DOF 移动器和插件

P6DOF 移动器: 这是 Warlock 中用于模拟平台运动的一个旧组件。它的相关插件负责将特定于 P6DOF 移动器的数据填充到平台详细信息中。  
弃用状态:P6DOF 移动器及其相关插件已被弃用，取而代之的是新的 SixDOF 移动器及其插件。因此，P6DOF 数据默认是禁用的，用户需要通过插件管理器启用它。

# SixDOF 移动器和插件

SixDOF 移动器: 这是一个更新的组件，提供更先进和灵活的六自由度运动模拟功能。它被设计为 P6DOF 的替代品，提供更好的性能和功能。  
SixDOF 数据: 取代了 P6DOF 数据，成为新的标准。它提供了更精确和详细的运动数据，适用于现代模拟需求。

# 迁移建议

对于使用 Warlock 进行模拟的用户，建议逐步迁移到 SixDOF 移动器和插件，以利用其增强的功能和支持。虽然 P6DOF 仍然可以通过插件管理器启用，但为了获得更好的性能和未来的支持，使用 SixDOF 是更好的选择。

# 5.2.2.5.5. 坐标转换工具 Position Converter Tool - Warlock

Position Converter Tool 用于在 Latitude/Longitude (LL) 和 Military Grid Reference System(MGRS) 格式之间进行转换。选择转换下拉框中的所需转换，并按照指定格式输入位置。有效输入将生成所需格式的结果。结果可以手动复制，或者可以使用“CopyResult”按钮快速将结果复制到剪贴板。

请注意，MGRS 是一种二维网格系统，用于唯一标识地球上任何地方的一个平方米。它基于 UTM（在 $\boldsymbol { 8 0 } ^ { \circ } ~ \mathsf { S }$ 和 $8 4 ^ { \circ } ~ \mathsf { N }$ 纬度之间）和 UPS 系统。

# 5.2.2.5.6. 投影器功能 Projector - Warlock

投影器功能允许在地图显示上显示传感器投影，以可视化瞬时传感器覆盖范围及其受地

形影响的情况。这种可视化工具帮助用户更直观地理解传感器的覆盖能力和限制。

![](images/3ba052e7e4c230936140ef93021ce15ca7aa1ea90fc979781ccb83d9bc79c389.jpg)

# 如何启用投影

平台上下文菜单: 通过右键点击平台并从平台的上下文菜单中选择相应选项来启用或禁用投影。此选项仅在以下条件满足时显示：

▫ 传感器已开启并处于活动状态。  
□ 传感器的模式具有关于传感器瞄准线对称的矩形视场。  
□ 视场的方位角和仰角不超过 135 度。

# 功能限制

地形数据库: 投影使用的是显示的视觉地形数据库，而不是模拟加载的地形。显示的精度受当前显示的细节级别影响。  
地形变化: 当加载的地形发生变化时，投影将隐藏。  
视场限制: 仅对具有关于瞄准线对称的矩形视场的传感器可见。方位角和仰角的视场不得超过 135 度。  
范围限制: 投影的范围从 100 米到 7000 公里。超出此范围的任何物体都不会影响投影，也不会被投影影响。  
更新频率: 投影每秒更新一次。

# 5.2.2.5.12. 自定义对话框工具 WsfPrompt - Warlock

WsfPrompt 是 Warlock 中用于创建自定义对话框的工具，允许用户通过 AFSIM 脚本在Warlock 中显示对话框。这些对话框可以配置按钮名称、每个按钮执行的脚本，以及对话框的标题和消息。

使用 WsfPrompt 的方法

创建对话框: 使用 WsfPrompt.Display 方法来创建对话框。

```txt
WsfPrompt.Display("title", "message", {"script_1", "script_2", ..., "script_n"}, {"button_1", "button_2", ..., "button_n"}) 
```

"title": 对话框的标题。  
"message": 对话框中显示的消息。  
{"script_1", "script_2", ..., "script_n"}: 每个按钮对应的脚本。如果某个按钮不需要执行脚本，可以使用空字符串。  
{"button_1", "button_2", ..., "button_n"}: 按钮的名称。   
暂停模拟: 如果需要，可以设置对话框在用户做出选择之前暂停模拟。

# 5.3. 复盘回放工具 Mystic

Mystic是一个用于可视化AFSIM模拟事件记录的工具。它在2.7.0版本中从“结果可视化”重命名为 Mystic。该工具通过读取 AFSIM 生成的事件记录（AER）文件，将模拟结果以地理上下文的方式展示给用户。Mystic 可以通过在 Wizard 中双击生成的 AER 文件启动，也可以作为独立应用程序运行。

# 功能和使用

可视化模拟结果:Mystic 专门用于展示 AFSIM模拟的结果，帮助用户在地理背景下理解和分析模拟数据。  
文件格式:Mystic 读取 AFSIM 事件记录文件（.aer），这些文件是模拟过程中生成的特定应用程序事件记录.

生成 AER 文件:Mystic 使用 AFSIM 模拟事件记录（AER）文件，这些文件是模拟过程中生成的二进制文件，类似于事件日志。

# 5.3.2.2.3.5. 抬头显 Head Up View - Mystic（弃用）

警告：HeadUpView 插件已被弃用且不再支持。用户应避免使用它，因为它将在不久的将来从 AFSIM 中移除。应使用 ACES Display 代替，它提供了许多相同的功能以及其他能力。

HeadUpView 显示一个窗口外（OTW）视图，其中包括一个通用的抬头显示（HUD）覆盖。该工具可通过在上下文菜单中右键单击平台后使用。如果需要，可以同时打开多个抬头视图。

注意：要显示所有可用数据，需要一个具有态势感知处理器的平台。

注意：HeadUpView 插件是一个原型功能，尚未完成。因此，它默认是禁用的，必须通过插件管理器启用。
![](images/c4074fdf68363bc5ccb13bb6ad25344444d24e03a247f73eb127324baba270c1.jpg)

概述

显示器由中心的 HUD 和其下方的简单前置控制（UFC）显示组成。

HUD 的顶部提供了一个水平航向带，当前航向（度）显示在带的中心框中。

HUD 的左侧显示地速（节）、校准空速（节）、迎角（alpha，度）、马赫数、当前过载和可用过载。它还包括 HUD 模式（如 NAV）以及箔条和照明弹消耗对策的计数。

HUD 的右侧显示垂直速度（英尺/分钟）、气压高度（英尺）、雷达高度（英尺），以及航路点信息，包括航路点编号、航向、航路点距离（海里）和到达航路点的时间（小时:分钟:秒）。它还包括导航模式和飞行员控制信息。

HUD 的底部显示滚动指示器。

中心显示俯仰/滚动梯和速度矢量。

UFC包括左侧的标准化推力指示器。绿色区域表示军用功率（干推力），红色区域表示加力/再热（湿推力）。中心显示质量摘要（所有值以千磅为单位），显示总重、总燃料、内部燃料和外部燃料。

注意：并非所有数据都可能显示，因为某些移动类型不提供所有显示的数据，但所有可用数据将被显示。
