# 5.2.2.2.3.2. 空中交战综合显示 ACES Display - Warlock

![](images/a5e32609ac888e4c2f85e660aa38afc942a79d6a17f34b51b4e6f208ba6483d4.jpg)

Air Combat Engagement Summary (ACES) Display 是一个集成显示工具，旨在为包含WSF_SA_PROCESSOR 的平台提供多组空战数据。它通过一个可重新配置的显示界面，整合

了多种空战信息。

# 功能概述

数据要求：要显示所有可用数据，需要一个具有态势感知处理器的平台。  
插件状态：ACESDisplay 插件是一个原型功能，尚未完成。因此，它默认是禁用的，必须通过插件管理器启用。  
打开显示：在地图显示或平台浏览器中为平台打开上下文菜单（右键单击），然后选择“ACES View”以打开显示。

ACESDisplay 提供了一个强大的工具，用于分析和可视化空战数据，尽管它仍处于开发阶段，但它为用户提供了丰富的战术信息和灵活的显示选项。

# 显示同步

在 Warlock 中，战术态势显示（TSD）和态势感知显示（SAD）的缩放和居中可以通过点击中央的“Display Sync”按钮来同步。当启用时，左侧区域的缩放和居中将作为同步显示的参考。

# 显示切换

在 Warlock 中，四个圆形显示区域可以通过点击顶部或底部显示的左侧或右侧来进行切换。这一功能允许用户根据需要重新排列显示内容，以便更好地组织和查看不同的数据集。

![](images/5225e718b7d0fac60cfb81fd8a5b9020c488232dae4443fcd071522105397dd8.jpg)

# 态势感知显示 Situation Awareness Display

![](images/c0afd3790fb5131d2e268d70344e123d5d0c84201122379cd65e230ee433d965.jpg)

态势感知显示（SAD）为用户提供了一个战术感知的图形化总结，利用平台的WSF_SA_PROCESSOR 数据。这种显示方式帮助用户理解环境中的实体及其动态变化，从而在空战环境中做出更有效的决策。

# 感知

在态势感知显示（SAD）中，感知到的实体根据其识别信息进行绘制。实体可能还包括其他信息，例如实体图标右侧的高度（以千英尺为单位），图标左上角的优先威胁编号，以及图标左下角的优先目标编号。实体的方向与其航向一致。被态势感知处理器（SAP）标记为重要的实体会显示为白色轮廓。漂移中的实体显示为透明，因此看起来较暗。如果设置了eyes_track_processor，则在实体图标的左侧会出现一个眼睛形状的图标。

# 群组

![](images/440bcbe683c9327490a7d1c9baf9d7314b4cef9d4b745e9a3ae8f3fb9df78380.jpg)

在态势感知显示（SAD）中，群组通过足够大的圆圈来显示，以包含群组的所有成员。

可以在 WSF_SA_PROCESSOR 中指定最小群组半径，这对于可视化非常有帮助。每个群组都有一个唯一的 ID，显示在群组圆圈的右上方。

群组颜色：

▫ 如果群组中的所有感知实体都被视为敌机/敌对（bandit/hostile），则群组以红色显示。  
□ 否则，群组将显示为黄色（bogie/unknown）。  
□ 如果群组仅包含漂移中的感知实体，则该群组将以淡化（幽灵）颜色显示。  
□ 被标记为“重要”的群组将被白色轮廓包围。

群组状态：

▫ 如果群组未聚焦，则其实体不会出现。相反，它将具有透明填充，并在中心显示实体数量。

所有感知到的实体都是某个群组的成员。然而，除非在 SAD 设置中关闭“显示感知”选项，否则仅包含一个实体的群组不会显示。

图标形状：

▫ 航空领域图标（包括飞机和武器）使用人字形。  
□ 其他领域使用方形图标。

# 战术态势显示（TSD）

![](images/a94048d94bb4067388f53487e010111456dab809d797fbdac148ab2045c23288.jpg)

战术态势显示（TSD）为用户提供当前“战术态势”的图形化总结，利用轨迹数据来呈现战场上的动态信息。这种显示方式帮助用户更好地理解和分析战术环境中的各种因素，从而做出更明智的决策。

TSD 系统通常集成了空中交通、天气、地形和特殊用途空域的信息，作为地面操作员或调度员的主要图形界面，支持单飞行员或减少机组人员操作的研究模拟。此外，TSD 还可以接收来自多个雷达源的数据，将其组织成马赛克显示，并在计算机屏幕上呈现。这种显示允许交通管理协调员选择和突出显示单个飞机或群组。

在军事应用中，TSD 通过地图显示跟踪所有友军和敌军的行动，使指挥官能够更好地了解战场动态，并据此做出行动决策。这种功能对于战术规划和态势感知至关重要。

# 威胁警告显示（TWD）

![](images/93d88b2dd05cca335841f00848e36d50e86e73622e5d5367494fdb606c7693ba.jpg)

威胁警告显示（TWD）用于显示电子支援措施（ESM）/雷达告警接收器（RWR）轨迹，并可能附带识别 ID。要将数据输入到此显示中，必须指定 esm_track_processor。根据轨迹的识别，不同的符号和颜色将被绘制。

符号和识别：

：默认 未知，方形，标识为

twdm：导弹，菱形，标识为‘M’

RWR 系统的主要任务是检测雷达系统发出的电磁信号，并在拦截到的雷达信号被其警告库分类为威胁时发出警告。这种警告可以用于手动或自动规避检测到的威胁。

通过 TWD，用户可以快速识别和响应潜在的威胁，从而提高战术态势感知和决策能力。

# 主飞行显示（PFD）

主飞行显示（PFD）提供类似于抬头显示器（HUD）的数据，帮助飞行员在飞行中获取关键的飞行信息。这些信息包括俯仰、滚转、校准空速（KCAS）、高度、迎角、垂直速度和过载（G-Load）。PFD 通常是现代飞机仪表的一部分，结合了传统模拟仪表的信息，通过液晶显示器或阴极射线管显示设备呈现.

# 5.2.2.3.3. 卫星任务序列调整工具 Astrolabe - Warlock

Astrolabe 是一个用于轻松创建和修改空间平台任务序列的工具。

# 功能概述

单平台操作：Astrolabe 一次只能处理一个平台，并可以修改该平台的任务序列。当将任务序列写入输入堆栈时，Astrolabe 首先尝试更新现有的任务序列。如果没有这样的序列，Astrolabe 会将任务序列添加到平台定义中的 mover 块的末尾。如果没有 mover块（例如，如果 mover 在平台的类型中定义，并且平台定义中不需要更改），则任务

序列将添加到平台定义的末尾。

最小化更改：Astrolabe 会尽可能少地更改现有任务序列。特别是，事件属性的单位不会被修改，除非用户明确更改它们。在某些情况下，Astrolabe 可能会改变输入命令的确切顺序。  
编辑器集成：当 Astrolabe将输入写入输入堆栈时，Wizard 中将打开 Astrolabe修改的文件的编辑器，并将修改记录到撤销/重做历史中，以便用户可以轻松撤销 Astrolabe所做的更改。

控件说明

14. 平台选择：提供两种选择平台的方法。用户可以在下拉框中选择平台，或者使用工具按钮从地图显示中选择平台。在这两种情况下，只能选择本地控制的空间平台。

# 任务序列验证

在 Astrolabe 中设计任务序列后，可以对序列进行验证。这一过程检查任务和任务事件的多个细节，并告知用户所设计的序列是否可执行。验证过程与在 AFSIM 模拟中初始化任务序列时的验证过程相同。这使用户能够快速检查序列构建中的许多错误，而无需运行完整的场景。

# 验证步骤

3. 输入检查：首先检查输入，以确保事件参数在允许的范围内。  
4. 标准验证：然后通过 AFSIM 的标准验证过程运行任务序列。

在第一步中出现的错误会通过对话框显示问题的概要。为了更容易地纠正问题，错误事件会被选中，以便用户立即定位到问题所在。如果单个事件有多个错误，这些错误会同时显

示。如果多个事件有错误，用户将仅收到序列中第一个具有超出范围事件参数的事件的通知，因此用户需要再次选择“验证”直到所有此类错误得到纠正。

一旦所有事件参数都在范围内，验证过程的结果将显示在序列面板中。序列中的每个条目将显示五个附加信息列，以及用于轨道事件预览的颜色键。附加数据列依次为：验证的总体状态、事件的 delta-v 成本、平台剩余的 delta-v 预算、事件的模拟时间，以及事件的日期和时钟时间。

# 验证状态

PASS：事件可以继续，所有前置和后置条件都满足，并且有足够的 delta-v来执行事件。  
FAIL：某些成功执行的条件不满足。将鼠标悬停在 FAIL 指示上可以查看失败的原因。  
Unverifiable：事件的执行依赖于模拟的实际状态，例如拦截事件需要知道目标平台的状态才能计算事件的细节。

如果任何事件的状态为 FAIL 或 Unverifiable，则所有后续事件将无法验证，因此不会为这些事件提供额外信息。

# 轨道事件预览

在任务序列验证后，可以在地图显示中可视化任务序列的预览。要查看这些预览，必须在序列面板中选择已尝试验证的事件。对于显示的每个事件，Astrolabe 将显示事件前后的轨道。显示事件的颜色将与序列面板中事件名称旁边指示的颜色匹配，事件前的轨道绘制得更暗。如果选择了一个序列，该序列中的所有事件将被预览。为了清楚地显示事件的平台位置，在轨道上放置一个楔形图标，使箭头指向特定事件的最终轨道。在某些情况下，事件前后的轨道是相同的（例如，分级事件），在这种情况下只显示一个轨道。

# 遗传算法参数

种群大小（PopulationSize）：设置每代评估的不同潜在解决方案或样本的数量。增加种群大小会扩大算法的搜索空间，但需要更多的计算时间。  
最大代数（MaximumGenerations）：设置算法执行的最大代数。增加代数为算法提供更多机会以获得更优化的解决方案，但也会增加计算时间。  
保留比例（Carry-OverFraction）：指定从一代到下一代保留的潜在解决方案的比例。较大的保留比例通过保留更多“适合”的样本使算法趋向于最优解，而较小的保留比例则扩大搜索空间，允许算法探索更多潜在解决方案。通常建议保留比例在 0.7 到 1.0 之间。  
交叉概率（Cross-OverProbability）：指定样本与种群中另一个样本“交换”特征的概率。第二个样本以“锦标赛”方式选择，即从随机选择的两个样本中选择更适合的一个。较高的交叉概率通过结合其他成功样本的特征来扩展算法的搜索，以达到最优解。通常建议交叉概率在 0.5 到 0.8 之间。

变异概率（MutationProbability）：指定样本的某个方面自发变化的概率。较高的变异概率扩大了算法的搜索空间，但可能会妨碍其快速收敛到良好解决方案的能力。通常建议变异概率在 0.01 到 0.10 之间。

# PID 控制器特定参数

Kp 范围：设置所选 PID 控制器的比例控制通道使用的最小和最大增益因子。较高的值可以促进快速的初始响应，但可能导致显著的超调。  
Ki 范围：设置所选 PID 控制器的积分控制通道使用的最小和最大增益因子。此值有助于减少稳态误差，但通常应小于 Kp。  
Kd 范围：设置所选 PID 控制器的微分控制通道使用的最小和最大增益因子。较高的值可以减轻超调，但如果信号表现出显著变化（看起来“嘈杂”或难以预测），也可能导致不稳定。  
命令运行时间（CommandRunTime）：设置遗传算法发送的每个命令执行的仿真时间（以秒为单位）。运行时间应使平台有足够时间稳定到目标状态以获得良好解决方案，但超过该时间将增加算法的运行时间而无益。  
误差权重（ErrorWeight）：设置遗传算法用于调谐的目标函数中总误差积分的权重因子。相对于超调权重增加此因子将更强调总误差，推动 PID 控制器快速达到目标状态，而对超调幅度的强调较少。  
超调权重（OvershootWeight）：设置遗传算法用于调谐的目标函数中超调项的权重因子。相对于误差权重增加此因子将更强调避免 PID 控制器内的超调，推动其达到可能较慢但超调较少的期望状态，以避免系统的不良行为。

# 调谐器最佳实践

虽然自动调谐过程经过逐步改进，但仍然不完美。在许多情况下，仅通过“调谐模型”按钮即可获得足够的结果。事实上，更新演示中使用的飞行器几乎无需手动干预即可调谐。然而，用户可以牢记一些事项，以提高调谐过程的效率。

# 考虑飞行器能力和限制

限制和设置块：调谐器基于限制和设置块定义的限制进行操作。最初，这个块由 MoverCreator 自动生成，仅基于飞行器是武器还是传统飞机。因此，默认设置可能不适合不太灵活的飞行器，因为调谐器可能会测试飞行器无法达到的命令，从而导致该控制通道的调谐质量较差。  
机体稳定性：考虑机体能够稳定的极限。例如，在调谐 B-FW 飞翼演示飞机时，必须在自动驾驶仪配置脚本中手动降低最大滚转率。飞翼设计在受控的倾斜转弯中自然不稳定，限制其命令滚转率有助于调谐器提供更强的自动驾驶仪。

# 调整 PID 控制器

CAS 和 Beta 通道：对于许多使用 CAS 的飞机，自动调谐 Beta 通道可以产生合理的侧滑响应，但在普通飞行中可能表现出高度振荡的行为。如果观察到这种行为，减少Beta 通道的 Kp 似乎效果很好。  
积分增益的影响：对于某些控制通道，调谐器可能允许积分增益占主导地位；对此结果要特别谨慎。调谐器使用阶跃输入测试 PID，对于这些输入，积分重的控制器可能是合理的，但在飞行中可能表现不佳。尤其要检查那些参与主动飞行控制阻尼的控制器，如

滚转、偏航和俯仰速率 PID，有时还包括 Alpha 和 BetaPID。可以通过限制最大累积或 Ki 本身到一个非常小的值来限制 Ki。

# 进阶调谐技巧

前馈表：自动生成的前馈表为完成命令所需的输入提供了知情的估计。在某种程度上，这用更量身定制的响应替代了一刀切的 Kp 增益。在某些情况下，如高过载导弹的Alpha 和 Beta 控制，前馈预测结合自然快速的系统响应可以允许非常小的 Kp。只有在系统响应不够快以满足任务需求时才“纠正”这一点。此外，如果系统在飞行中表现出过多的不稳定性，考虑将这些通道的 Kp 完全限制为 0。  
调整 Kd 值：在某些情况下，如导弹的滚转速率控制，调谐器会建议非常小的 Kd 值，但仍会产生一些抖动的控制器。在这种情况下，考虑进一步减少 Kd，趋向于 0。

# 手动调整建议

实验参数：如果时间允许，尝试更高的交叉或变异率和更大的种群规模。早期成功的调谐可以节省后续的时间和成本，因此在调谐过程中投入一些时间是值得的。  
手动调整值：调谐器在对数空间中搜索，因为可能的飞行器配置是无限的，我们需要有效地覆盖地面，特别是在接近 0 的地方。手动调整值时也要这样做。调谐器使用 10 为底，但二分搜索策略也是完全合理的——关键是在起点完全不可接受时进行大幅度调整，而在接近有用的解决方案时进行小幅度调整。  
重新评估相关控制器：记住导航控制器正在与低级控制器协同工作并进行补偿。例如，如果需要调整飞机的滚转速率，也应重新评估依赖于它的控制器，如倾斜角和滚转航向。

平台选择：平台选择框显示场景中当前存在的所有 SixDOF 飞行器，允许用户选择要观察和调谐的飞行器。

PID 增益：显示并可直接修改比例、积分和微分增益因子（分别为 Kp、Ki 和 Kd），以操控平台自动驾驶仪的行为。

高级过滤器：高级过滤器选项卡显示所有其他可用的方法来控制或限制 PID 控制器输出。可用选项包括：

□ 低通滤波器 Alpha：修改在输出到控制器之前对原始微分信号进行的滤波。这有助于减少由于系统变化率估计中的数值不精确性导致的输出噪声。值为 1.0 表示完全原始的微分信号，0.0 表示完全无响应的微分信号。  
□ 最大累积：限制 PID 控制器生成的积分信号的最大值，以防止积分饱和导致系统不稳定。  
▫ 限制：显示当前选择的命令链中任何 PID 设置的所有限制。这些限制不能在调谐器对话框中修改，因为它们是在平台定义中设置的。

预置位置：允许用户指定平台状态以便使用高度和速度轻松返回。按下“设置状态”按钮后，所选平台将被发送到指定状态。  
冻结命令：允许用户冻结所选飞行器状态的属性，以隔离正在调谐的单个行为。可冻结的属性包括高度、速度、位置、燃料水平、滚转、俯仰和偏航。  
连续命令：允许用户指定并发送命令到所选 PID 控制器。这些命令将在按下相应的“发送命令”按钮后由平台执行，并可以在右侧的图视图中实时观察。  
时间管理命令：允许用户指定并发送单个命令到所选 PID 控制器。按下“执行”按钮时，所选平台被发送到预置状态并给出指定命令。仿真以最大速度运行，直到仿真中指定的时间过去。  
平台状态图：平台状态图（顶部）显示由当前观察的 PID 控制器控制的平台属性的当前状态，以及该属性的当前命令状态。  
PID 控制器信号图：PID 控制器信号图（底部）显示当前由正在观察的 PID 控制器发出的比例、积分、微分和前馈信号的当前值。  
自动调谐 PID：利用遗传算法自动调谐当前观察的 PID 控制器。建议用户即使在 PID控制器自动调谐后也要查看平台响应，因为手动调整 PID 增益可能仍然是实现自动驾驶仪可接受行为的理想选择。  
自动调谐模型：自动调谐当前选择的平台使用的所有 PID 控制器。  
清除图：清除图中当前显示的所有数据。  
导出：保存当前的 PID 增益值状态，以便将来与已调谐的模型一起使用。

# 遗传算法参数

种群大小（PopulationSize）：设置每代评估的不同潜在解决方案或样本的数量。增加种群大小会扩大算法的搜索空间，但需要更多的计算时间。  
最大代数（MaximumGenerations）：设置算法执行的最大代数。增加代数为算法提供更多机会以获得更优化的解决方案，但也会增加计算时间。  
保留比例（Carry-OverFraction）：指定从一代到下一代保留的潜在解决方案的比例。较大的保留比例通过保留更多“适合”的样本使算法趋向于最优解，而较小的保留比例则扩大搜索空间，允许算法探索更多潜在解决方案。通常建议保留比例在 0.7 到 1.0 之间。  
交叉概率（Cross-OverProbability）：指定样本与种群中另一个样本“交换”特征的概率。第二个样本以“锦标赛”方式选择，即从随机选择的两个样本中选择更适合的一个。较高的交叉概率通过结合其他成功样本的特征来扩展算法的搜索，以达到最优解。通常建议交叉概率在 0.5 到 0.8 之间。  
变异概率（MutationProbability）：指定样本的某个方面自发变化的概率。较高的变异概率扩大了算法的搜索空间，但可能会妨碍其快速收敛到良好解决方案的能力。通常建议变异概率在 0.01 到 0.10 之间。

# PID 控制器特定参数

Kp 范围：设置所选 PID 控制器的比例控制通道使用的最小和最大增益因子。较高的值可以促进快速的初始响应，但可能导致显著的超调。

Ki 范围：设置所选 PID 控制器的积分控制通道使用的最小和最大增益因子。此值有助于减少稳态误差，但通常应小于 Kp。  
Kd 范围：设置所选 PID 控制器的微分控制通道使用的最小和最大增益因子。较高的值可以减轻超调，但如果信号表现出显著变化（看起来“嘈杂”或难以预测），也可能导致不稳定。  
命令运行时间（CommandRunTime）：设置遗传算法发送的每个命令执行的仿真时间（以秒为单位）。运行时间应使平台有足够时间稳定到目标状态以获得良好解决方案，但超过该时间将增加算法的运行时间而无益。  
误差权重（Error Weight）：设置遗传算法用于调谐的目标函数中总误差积分的权重因子。相对于超调权重增加此因子将更强调总误差，推动 PID 控制器快速达到目标状态，而对超调幅度的强调较少。  
超调权重（OvershootWeight）：设置遗传算法用于调谐的目标函数中超调项的权重因子。相对于误差权重增加此因子将更强调避免 PID 控制器内的超调，推动其达到可能较慢但超调较少的期望状态，以避免系统的不良行为。
